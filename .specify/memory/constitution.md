# Mecris Constitution

## Core Principles

### I. Reality Enforcement
Mecris is not a savior; it is a witness. If the human fails, the system must record that failure with clinical precision. We do not forge "0.0 heartbeats" to prevent derailment. The Ghost Archivist ensures the internal record (Neon DB) matches reality, accepting that accurate derailments are a valid outcome.

### II. The Fleet (Distributed Consciousness)
We operate across specialized modalities:
*   **The Vanguard (Python MCP)**: Rapid logic iteration and strategic synthesis.
*   **The Bridge (Android)**: Native telemetry collection (Health Connect).
*   **The Brain (Spin WASM)**: Consolidated, language-agnostic logic core.
*   **The Executor (Claude Bot)**: Autonomous GitHub Actions sessions (The Gall Loop).

### III. Test-Driven Generation (TDG)
All code changes MUST be verified by automated tests BEFORE and AFTER implementation. We follow the Red-Green-Refactor cycle strictly. A feature is not complete until its behavioral correctness is empirically proven in the target environment.

### IV. Logic Vacuuming
Following the "Useless Architecture" pattern, business logic is developed in Python (The Vanguard) and then "vacuumed" into the WASM Brain using `componentize-py`. This prevents code duplication and ensures a single source of truth for calculations like the Review Pump.

### V. Budget Stewardship
We treat LLM tokens as a finite resource. The system must monitor spend across providers (Anthropic, Groq) and guide the LLM to provide maximum value (strategic insight vs. simple logging).

## Feature Domains

### The Nag Ladder
A multi-tier notification system (Tier 1: Gentle, Tier 2: Escalated, Tier 3: Emergency) that uses idle-time detection and runway urgency to drive human behavior without being ignored.

### The Majesty Cake
A daily aggregate status widget (X/Y goals) that provides a unified "All Clear" celebration. It rewards completion across all modalities (Steps, Arabic, Greek).

### The Ghost Archivist
A deterministic, non-LLM background process that synchronizes external APIs (Clozemaster, Beeminder) with the internal Neon DB to enforce reality at the day boundary.

## Governance
*   This Constitution supersedes all other documentation.
*   Amendments require a formal session log entry and a version bump.
*   All PRs from `mecris-bot` must be reviewed against these principles by the Strategic Synthesizer (Gemini).
