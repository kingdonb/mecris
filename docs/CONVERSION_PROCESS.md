## Workflow Overview
1. **Document Analysis**
   - Scan each target file for markdown task items (`- [ ]`).
   - Record tasks that have **more than five** items; smaller files can be handled manually.
2. **Issue Creation**
   - For each task item:
     - Derive a concise title (first few words of the description).
     - Choose a label based on the task type (e.g., `documentation`, `enhancement`, `bug`).
     - Populate the Issue body with:
       - Original task description.
       - A reference link to the source file and line number.
   - Use the GitHub API (`POST /repos/:owner/:repo/issues`).
3. **Document Cleanup**
   - Replace the original markdown task line with a reference to the newly created Issue, e.g.:
     ```markdown
     - [ ] Implement rate limiting for API endpoints <!-- #123 -->
     ```
   - If a task is no longer needed, simply delete the line.
4. **Verification**
   - Run a script that confirms every `<!-- #<num> -->` comment maps to an existing Issue.
   - Ensure no remaining `- [ ]` items remain in the processed documents.
